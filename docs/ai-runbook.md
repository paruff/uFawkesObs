# AI Capability Alerts — Runbook

> Responding to AI capability alerts from Prometheus.
> All alerts have `category: ai-capability` label.

---

## AILLMLatencyHigh

**Severity:** warning
**Expression:** `ai:llm_request_latency_p99:seconds > 10` for 5m

**What it means:** The p99 LLM response time has been above 10 seconds for 5 minutes.

**Triage steps:**
1. Check which model/provider is experiencing high latency: query `ai:llm_request_latency_p99:seconds by (gen_ai_system, gen_ai_request_model)`
2. Verify provider status (OpenAI, Anthropic, etc.) for known outages
3. Check if rate limiting or quota throttling is active
4. Consider model downgrade (e.g. GPT-4 -> GPT-4-mini) for non-critical paths

**Resolution:**
- If provider outage: wait for recovery, switch to fallback provider
- If rate limited: reduce request concurrency or upgrade quota
- If model too slow: switch to faster model variant

## AILLMLatencyHighAbsent

**Severity:** warning
**Expression:** `absent(ai:llm_request_latency_p99:seconds)` for 5m

**What it means:** The LLM latency recording rule is not producing any data. AI SDK telemetry may not be flowing.

**Triage steps:**
1. Check that AI SDKs are instrumented and sending OTLP to the collector
2. Verify OTel Collector `metrics/ai` pipeline is receiving data
3. Check Prometheus scrape targets for `otel-collector:8889`

## AIReworkRateHigh

**Severity:** warning
**Expression:** `(dora:rework_rate:ratio or vector(0)) > 0.10` for 7d

**What it means:** The AI rework rate (code generated by AI that was subsequently modified/fixed) has been above 10% for 7 days. This is the DORA 2025 watch threshold.

**Triage steps:**
1. Review which services/teams have high rework rates
2. Audit recent AI instruction quality in AGENTS.md files
3. Check if prompts need updating for recent model changes

## AIReworkRateCritical

**Severity:** critical
**Expression:** `(dora:rework_rate:ratio or vector(0)) > 0.20` for 7d

**What it means:** The AI rework rate has been above 20% for 7 days. This is the DORA 2025 stop-features threshold. AI-generated code is creating more problems than it solves.

**Triage steps:**
1. STOP all new feature development using AI generation
2. Review AGENTS.md and PROMPT_LIBRARY.md for each affected service
3. Identify common failure patterns in AI-generated code
4. Fix AI instructions before resuming feature work
5. Re-run acceptance criteria after fixes to confirm improvement

**Resolution:**
- Update AGENTS.md with clearer constraints and examples
- Add test-first requirements forcing higher quality output
- Reduce AI scope until rework rate drops below 10%

## AITokenBudgetHigh

**Severity:** warning
**Expression:** `ai:token_usage_rate:per_minute > 100000` for 5m

**What it means:** Token consumption has exceeded the budget threshold of 100,000 tokens per minute for 5 minutes.

**Triage steps:**
1. Check which services are consuming tokens: query `topk(5, sum(rate(app_metrics_gen_ai_client_token_usage_sum[5m])) by (service))`
2. Review if a new feature or integration was recently deployed
3. Consider rate limiting or model downgrade
